内部类、map、list、set 小记

发表于 2017-03-23

今天内部类的学习，看的比较蛋疼，一些概念看的似懂非懂，总结几点，更多关于内部类的总结还要完全掌握再说。

内部类的定义：可以将一个类的定义放在另一个类的定义内部，这就是内部类。

1.内部类可以访问外围对象的所有成员、以及所有元素，而不需要任何特殊条件。

public class Sequence {
    private Object[] items;
    private int  it;

    public Sequence(int size){
        items = new Object [size];
    }

    class abc{
        private void next(){
            //items
            //it 
            //此处只举例不再写详细代码叙述
        }
    }
}

2.当内部类需要生成对外部类对象的引用，可以使用外部类的名字后面紧跟圆点和this

public class DotThis{
    class Inner{
        public DotThis outer(){
            return DotThis.this;
        }
    }
}

3.当需要创建某个内部类的对象，你必须在 new表达式中提供对其他外部类的引用
这就需要 .new语法 如下

public class DotNew{
    public class Inner{}
    public static void main(String[] args) {
        DotNew dn = new DotNew();
        DotNew.Inner  dni = dn.new Inner();
    }
}

必须使用外部类的对象来创建该内部类对象，在拥有外部类对象之前是不可能创建内部类对象的，除非是静态内部类，它就不需要外部类对象的引用。

内部类先记录到这里，到时再开章节讲。

Java中提供了一套相当完整的容器类。
其中基本类型是List、Set、Queue和Map,这些对象类型也被称为集合类

在容器类中，Set对于每个值只保存一个对象，Map是允许你将某些对象与其他一些对象关联起来的关联数组，java容器类可以自动调整自己的尺寸。

List的主要实现类有：ArrayList，LinkedList
ArrayList 它长于随机访问元素，但是在List的中间插入和移除元素时较慢，
LinkedList 它通过代价较低的在List中间进行插入和删除操作，提供了优化的顺序访问，LinkedList在随机访问方面较慢，但是它的特性集较ArrayList更大。
两种List都是按照被插入的顺序保存元素。

下面三种Set相同点，每个相同的项只有保存一次
HashSet:使用相当快的获取元素方式，因此存储顺序没有实际意义。
TreeSet: 按照比较结果的升序保存对象，
LinkedHashSet:按照添加的顺序保存对象。

三种基本风格的Map:HashMap,TreeMap,LinkedHashMap
与HashSet一样，HashMap也提供了最快的查找技术，也没有任何明显顺序保存其元素，
TheeMap 按照比较结果的升序来排序，LinkedHashMap按照插入的顺序保存键，同时还保留了HashMap的查询速度。
