2.Activity、Fragment的生命周期与四种启动模式
activity:
oncreate-> onstart -> onResume -> onpause -> onstop - > ondestroy
当activity 停止 并没销毁时，调用onRestart
启动模式：分别为 standard、 singleTop 、singleTask、 singleInstance
singleTop:栈顶复用模式，当一个activity设置为这个模式后，启动时会先判断栈顶是否是当前activity有就不会重新创建，直接调用onNewIntent方法，当不是在栈顶就会重新创建。
singleTask:栈内复用模式，设置为此模式后，会专门给activity开一个栈，独立运行，不管重复创建多少次都只有一个，只是重新调用onNewIntent，在这个栈里如果不是栈顶，会直接把它放到栈顶，中间的activity会被移除。
singleInstance:单实例模式,是一种加强的singleTask模式，跟singleTask一样的特性，还加强了当activity设置此模式后，会独立运行在一个栈中，并且栈内不能有除此之外的Activity存在。
standard:默认的标准模式，直接启动一个activity，不管存在多少个。
Fragment:
onAttach->onCreate->onCreateView -> onActivityCreated ->onStart->onResume->onpause->onStop->onDestroyView -> onDestroy
->onDetach
attach：作用是与activity关联。
开发过程中主要是用onCreateView，

4.View的绘制流程
view的工作流程，主要是指measure,layout,draw这三大流程，
其中measure确定View的测量 宽/高,layout确定View的最终宽/高和这四个顶点的位置，而draw则将View绘制到屏幕上。
measure:measure在自定义中重写onMeasure把测量后的值通过setMeasuredDimension(宽，高),
layout:会通过setFrame设置四个顶点的位置，即初始化左右上下四个值，view的四个顶点就可以确定它在父容器中的位置，接着会调用子类的onLayout()这方法是父容器确定子元素的位置
draw:先绘制背景，再绘制自己，再绘制 children（子view）,再绘制装饰。

6.图片加载机制
1.图片加载首先会对bitmap进行一个压缩，压缩过程需要对bitmap设置一个缩放比率，缩放比率的计算通过，目标View接受的宽高，图片的宽高来计算，
首先判断图片宽高是否大于View的宽高，是就除以缩放比率，然后继续判断是否大于，缩放比率在每一次循环过程中都剩以2，为什么剩以2，是因为倍数缩放效果更稳定，
