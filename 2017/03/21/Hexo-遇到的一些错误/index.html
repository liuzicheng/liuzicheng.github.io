Hexo 遇到的一些错误

发表于 2017-03-21

从头开始说起吧，配置hexo时遇到的错误

1.先从SSH的配置说起 

  1.1 打开git bash 输入下面命令
ssh-keygen -t rsa

这里会要求你输入一次名称，两次密码(这个密码应该是不需要和Github重名的)    

1.2
  配置完后到 
   c:\users\name\.ssh 
  下找到 id_rsa.pub 文件 把内容复制

  登陆Github，找到Settings --> SSH and GPG keys --> New SSH key

  输入标题名称随意，复制过来的内容粘贴再这里。

1.3 
  测试是否配置成功
  输入
  ssh -T git@github.com   

  当提示以下内容表示配置成功！    

  You've successfully authenticated, but GitHub does not provide shell access.     

  另外初次使用还需要配置 username ,user-email

  git config --global user.name "[Your Name]"

  git config --global user.email "[Your Email]"

2.配置 _config.yml

当这一步时请别用
    https://github.com/UserName/UserName.github.io.git
因为，我就没成功过；
用下面的可以行的。记得替换成你自己的名字
type: git
  repo: git@github.com:UserName/UserName.github.io.git
  branch: master

3.error: failed to execute prompt script (exit code 1)；

   1.删除你hexo下面的.deploy_git文件夹

2.git config --global core.autocrlf false

3. hexo clean

4. hexo g

5. hexo d

以上就是我使用hexo 遇到的一些错误，哦对了，要记得在github中

new repository 刚开始没再github创建库，报了些错误。。。

用的编辑器是Sublime Text 不知道样式怎么样。
